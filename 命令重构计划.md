# 命令重构计划 ✅ 已完成

> 对标 [Playwright CLI](https://github.com/microsoft/playwright-cli) 命令设计，重构 wx-devtools-cli 的命令命名和功能覆盖。
>
> **状态：已完成** — 所有命令已重命名为 kebab-case 短动词，新增 5 个命令（hover, press, drag, storage, scroll），IDE 命名空间从 `ide` 改为 `ci`。

## 一、设计原则

1. **短动词优先** — 命令名尽量简短，LLM token 友好（`snapshot` 而非 `get_page_snapshot`）
2. **kebab-case** — 统一使用 kebab-case（`go-back` 而非 `navigate_back`），与 Playwright CLI 一致
3. **位置参数** — 高频参数放第一位，省略 flag 名（`click e21` 而非 `click --uid e21`）
4. **保留差异化优势** — 断言、诊断、IDE 管理等小程序独有能力继续保留

## 三、建议新增的命令

| 建议命令 | Playwright 对应 | 优先级 | 说明 |
|---------|----------------|--------|------|
| `hover <ref>` | `hover <ref>` | 中 | 使用 element.longpress() |
| `press <key>` | `press <key>` | 高 | 当前只能通过 `fill` 或 `eval` 间接实现，键盘事件是高频需求 |
| `drag <from> <to>` | `drag <start> <end>` | 低 | 通过 touch 事件链模拟，使用场景较少 |
| `storage` | `localstorage-*` | 高 | 通过 miniProgram.callWxMethod(method: string, ...args: any[]): Promise<any> 调用 `getStorageSync/setStorageSync/removeStorageSync/clearStorageSync` |
| `scroll <ref>` | `mousewheel` | 中 | 页面滚动，通过 `miniProgram.pageScrollTo(scrollTop: number): Promise<void>` 实现 |

## 四、不需要对齐的 Playwright 命令

| Playwright 命令 | 不适用原因 |
|----------------|-----------|
| `go-forward` | 小程序页面栈单向，无"前进" |
| `pdf` | 小程序无 PDF 导出 |
| `resize` | 窗口尺寸由设备模拟器决定 |
| `tab-list/new/close` | 小程序 Tab 是 TabBar 固定页，不可动态创建/关闭 |
| `cookie-*` | 小程序无 cookie |
| `localstorage-*` / `sessionstorage-*` | 对应我们的 `storage` 新命令，但 API 不同 |
| `state-save/load` | 小程序无法序列化 session 状态 |
| `dialog-accept/dismiss` | 小程序 Modal 通过 `wx.showModal` API 控制，无浏览器原生 dialog 事件 |
| `upload` | 小程序文件上传通过 `wx.chooseImage` 等 API |
| `tracing-start/stop` | miniprogram-automator 无 tracing |
| `video-start/stop` | 无录屏 API |
